## Description

Constructor Auth - NestJS backend with PostgreSQL database for authentication service.

## Features

- ✅ NestJS framework
- ✅ PostgreSQL database with TypeORM
- ✅ Docker Compose for database
- ✅ ESLint + Prettier configured
- ✅ Pre-commit hooks with Husky + lint-staged
- ✅ TypeScript strict mode

## Project setup

1. Install dependencies:

```bash
npm install
```

2. Set up environment variables:

Create `.env` file in the root directory (you can use `.env.example` as a template):

```env
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=postgres
DB_DATABASE=constructor_auth

PORT=3000
NODE_ENV=development
```

3. Start PostgreSQL database with Docker Compose:

```bash
docker-compose up -d
```

4. Run the application:

```bash
# development mode (watch mode)
npm run start:dev

# production mode
npm run start:prod
```

## Pre-commit hooks

This project uses Husky and lint-staged to automatically:
- Run ESLint on staged files
- Format code with Prettier
- Prevent commits with linting errors

## Database

PostgreSQL database is managed via Docker Compose. To manage the database:

```bash
# Start database
docker-compose up -d

# Stop database
docker-compose down

# Stop and remove volumes (clean database)
docker-compose down -v
```

The database will be available at `localhost:5432` with default credentials from docker-compose.yml.
